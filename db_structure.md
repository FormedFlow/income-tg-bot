Структура БД для ТГ бота по финансам

## Роли
1. Пользователь
2. Администратор

## Полномочия
### Пользователь
1. Управлять локальными категориями расходов/доходов
2. Добавлять доходы/расходы
3. Добавлять регулярные доходы/расходы (фиксированные)
4. Просматривать и удалять локальные доходы/расходы
5. Редактировать, удалять доходы/расходы
6. Просматривать статистику доходов/расходов за обозначенный период времени
7. Просматривать инфографику по категориям доходов/расходов


### Администратор
1. Управлять категориями доходов/расходов пользователей
2. Управлять статьями доходов/расходов пользователей
3. Управлять учетками пользователей


### Таблицы
1. user - таблица со всеми пользователями бота, имеет флаг is_admin.
2. transaction - транзакции, расходы, доходы. и флаг type для расходов/доходов.
3. category - категории доходов и расходов вместе.
4. recurring_transaction - регулярные платежи/поступления.
